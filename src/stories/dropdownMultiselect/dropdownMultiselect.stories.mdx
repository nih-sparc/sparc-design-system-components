# Dropdown Multiselect

## Properties
The dropdown multiselect component accepts the following properties:

`category`: The category associated with the dropdown multiselect. Object. Required. The category object must have a label, id, and data property assigned. 
Any nested parent data must have a children property where their childrens label and id are assigned.

`enabled`: When set to false, disallows the user from interacting with the dropdown. Boolean. Optional. Default value is true.

`collapseByDefault`: When set to true, collapses the dropdown by default. Boolean. Optional. Default value is false.

`defaultCheckedIds`: The ids of any nodes that should be checked by default. Array. Optional.

`visibleData`: The data that you want to be shown. Object. Optional. Setting this property is not reccomended for normal use cases and is available for the
sole purpose of allowing the sparc portal's 'data' and 'models and simulations' tabs share a dropdown. If not set, all the data will be shown as expected.
The visible data must have the categories id as a property on the object. Inside this object you can then specify the label of every selection that should be visible as a property of the object.

`tooltip`: The tooltip associated with the dropdown category. String. Optional. Setting this will cause the help icon to appear next to the category title and hovering over it will show the tooltip.

## Events
`selection-change`: Will fire when a user makes a selection change. Returns an array of selections.

### One Option Dropdown Multiselect

<Story name="One Option">{stories.OneOption()}</Story>

#### Code

```js
data() {
  return {
    category: {
      label: 'One Option',
      id: '0',
      data: [
      {
        label: 'One',
        id: '1',
      }]
    },
  }
}
```

```html
<dropdown-multiselect
  :category="category"  
/>
```
